# 싱송 다이어리 — 디자인 요구사항

외주 디자이너용 기능 명세. 비주얼 방향은 디자이너 재량.

---

## 앱 개요

일기를 쓰면 AI가 음악을 만들어주는 웹앱. 유저가 일기를 작성하고 음악 스타일 옵션을 선택하면, 30초짜리 음악이 자동 생성된다. 생성된 음악은 일기와 함께 저장되어 다시 들을 수 있다.

**플랫폼:** 웹 (데스크톱 우선, 모바일 반응형)
**언어:** 한국어 전용

---

## 화면 목록

### 1. 로그인

- 앱 이름 "싱송 다이어리"
- 앱의 한줄 설명 (예: "일기를 쓰면, 음악이 됩니다")
- Google 소셜 로그인 버튼
- 비회원은 접근 불가 (모든 페이지는 로그인 필수)

### 2. 메인 — 일기 작성

이 앱의 핵심 화면. 일기를 쓰고 음악 옵션을 고른 뒤 생성을 요청하는 곳.

**상단 바 (모든 화면 공통)**
- 앱 이름 (클릭 시 메인으로)
- 유저 프로필 (아바타 또는 이니셜)
- 로그아웃 버튼

**일기 작성 영역**
- 오늘 날짜 표시 (예: "2026년 4월 7일 월요일")
- 텍스트 입력 영역. 충분히 넓어야 함. 플레이스홀더: "오늘 하루는 어땠나요?"

**음악 옵션 선택 영역**
- 섹션 제목: "음악 설정"
- 5개 옵션 그룹. 각 그룹마다 라벨 + 선택지 나열:

| 그룹 | 선택지 |
|------|--------|
| 장르 | 팝, 재즈, 클래식, 일렉트로닉, 앰비언트, R&B, 록, 어쿠스틱, 힙합, 보사노바 |
| 주요 악기 | 피아노, 기타, 바이올린, 첼로, 신스, 드럼, 플루트, 색소폰, 우쿨렐레 |
| 템포 | 아주 느리게, 느리게, 보통, 빠르게, 아주 빠르게 |
| 분위기 | 평화로운, 우울한, 신나는, 몽환적, 따뜻한, 쓸쓸한, 밝은, 긴장되는 |
| 질감/특징 | 미니멀, 레이어드, 오케스트라, 로파이, 어쿠스틱, 일렉트릭, 보컬 포함, 인스트루멘탈 |

- 각 그룹에서 1개씩 선택 (단일 선택)
- 선택/미선택 상태가 시각적으로 구분되어야 함
- 선택하지 않아도 제출 가능 (AI가 일기 내용 기반으로 판단)

**제출 영역**
- "음악 만들기" 버튼. 화면에서 가장 눈에 띄는 요소.
- 버튼 아래 또는 근처에 일일 잔여 횟수 표시: "오늘 남은 횟수: 2/3"
- 일기 텍스트가 비어있으면 버튼 비활성화

### 3. 생성 중 (로딩 상태)

일기 제출 후 음악이 만들어지는 동안 보여주는 화면. 보통 20~30초 소요.

- 제출한 일기 텍스트 (읽기 전용으로 표시)
- 선택한 옵션 요약 (예: "재즈 · 피아노 · 느리게 · 몽환적 · 로파이")
- 생성 중임을 나타내는 애니메이션. 음악 앱답게 사운드 웨이브나 파형 같은 모션이면 좋겠지만, 디자이너 판단에 맡김.
- 상태 텍스트: "음악을 만들고 있어요..."
- 예상 시간 안내: "보통 20~30초 정도 걸려요"
- "취소" 버튼 (부차적 스타일)

### 4. 생성 완료

음악이 완성된 후 결과를 보여주는 화면.

**일기 + 결과 영역**
- 일기 텍스트 (읽기 전용)
- 선택한 옵션 태그
- 생성 완료 상태 표시

**음악 플레이어**
- 재생 / 일시정지 버튼
- 현재 시간 / 전체 시간 표시 (예: "0:12 / 0:30")
- 재생 진행률을 시각적으로 표현 (프로그레스 바, 웨이브폼 등 형태는 자유)
- 다운로드 버튼

**추가 액션**
- "다른 스타일로 다시 만들기" 버튼 → 메인 화면으로 돌아가되 같은 일기 텍스트 유지, 옵션만 다시 선택
- "새 일기 쓰기" 버튼 → 메인 화면 (빈 상태)

### 5. 생성 실패

- 일기 텍스트 + 옵션 태그 (동일)
- 에러 메시지: "음악 생성에 실패했어요"
- "다시 시도" 버튼 (같은 내용으로 재요청)
- "돌아가기" 버튼 → 메인

### 6. 히스토리 (지난 기록)

과거에 작성한 일기와 생성된 음악을 다시 볼 수 있는 목록.

**목록**
- 카드형 리스트. 각 항목:
  - 날짜 (예: "4월 6일 일요일")
  - 일기 본문 발췌 (1~2줄, 넘으면 말줄임)
  - 음악 옵션 태그 (장르 · 악기 · 분위기 등)
  - 재생 버튼 (카드 안에서 바로 재생 가능)
- 최신순 정렬
- 카드 클릭 시 해당 일기의 상세 보기 (화면 4와 동일한 레이아웃)

**빈 상태**
- 아직 일기가 없을 때: "아직 기록이 없어요. 첫 번째 일기를 써보세요!" + 메인으로 이동 버튼

**일일 제한 초과 상태**
- 히스토리는 정상 표시
- 메인에서 "음악 만들기" 버튼 비활성화 + "오늘 횟수를 모두 사용했어요. 내일 다시 만들 수 있어요."

## 네비게이션 구조

```
로그인 → 메인 (일기 작성)
              ↓ 제출
         생성 중 → 생성 완료 (성공) / 생성 실패
              
메인 ←→ 히스토리 (상단 바에서 전환)
```

- 상단 바에 "메인"과 "지난 기록" 탭 (또는 네비게이션 요소)
- 현재 위치 표시

---

## 반응형 고려사항

- 데스크톱: 메인 화면의 일기 영역과 옵션 영역을 나란히 배치 가능
- 모바일: 일기 영역 아래에 옵션 영역 세로 배치. 옵션 그룹이 많으니 스크롤 가능하게.
- 음악 플레이어는 모든 기기에서 사용하기 편해야 함 (터치 영역 충분히)

---

## 상태 정리

각 화면에서 표현해야 하는 상태:

| 상태 | 설명 | 표시 위치 |
|------|------|----------|
| 미작성 | 일기 텍스트 비어있음. 제출 버튼 비활성화. | 메인 |
| 작성 중 | 텍스트 입력 중. 제출 가능. | 메인 |
| 제한 초과 | 일일 3회 모두 사용. 제출 불가. | 메인 |
| 생성 중 | 음악 생성 대기. 20~30초. | 생성 중 화면 |
| 생성 완료 | 음악 재생 가능. | 완료 화면 |
| 생성 실패 | 에러. 재시도 가능. | 실패 화면 |
| 기록 없음 | 히스토리 빈 상태. | 히스토리 |

---

## 참고

- 1개 일기에 여러 번 음악을 생성할 수 있음 (옵션을 바꿔서). 히스토리에서는 같은 일기에 여러 음악이 붙을 수 있음.
- 음악 길이는 30초 고정.
- 일일 생성 제한: 3회.
